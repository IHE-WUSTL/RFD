RFD 0-11100

Special Instructions

Your Form Filler under test will need to be configured for this test to return a form with Form ID: RFD 0-11100 The URL of the Form Receiver simulator is (you need to fill this in). No pre-population data is supplied.


Description

This test requires that the Form Manager responds to an ITI-34 transaction with the following data:

  1. Pre-population data is null
  2. workflowData.formID: RFD 0-11100
  3. workflowData.encodedResponse: false, meaning the Form Filler is requesting the URL of a form that can be opened in a web browser
  4. workflowData.archiveURL: empty string
  5. workflowData.context: null
  6. workflowData.instanceID: null

  • The Form Filler submits the Request for Form (ITI-34) using this value for formID: RFD 0-11100
  • The Form Manager simulator returns the URL of a form.
  • The Form Filler submits the form to the Form Receiver simulator.
  • The test system evaluates the request for all required properties (both positive and negative).
  • The testing system examines the Form Receiver simulator for successful submission of the form. The testing system retrieves the form that has been submitted and displays the values of all data elements in the submitted format.


Evaluation

  1. Open the URL returned by the Form Manager under test in a modern web browser. Make sure that you can successfully open the form and complete the data entry.
  2. When you submit the form, make sure your web browser reports no errors. Specifically, the form should be submitted to the Form Receiver simulator.
  3. Use the testing system to find the form data submitted to the Form Receiver simulator. Display the data element/value pairs and check for consistency with the values you entered.


Test Steps

  1. Send a Retrieve Form request to your Form Manager using the Submit Form Request button below.
  2. After you system returns the URL of a form, that URL should be displayed below under the button for submitting the form. Copy/paste that URL in a Web Browser.
  3. Complete the form and submit. The form should be transmitted to the Form Receiver simulator.
  4. View the submitted form by using the View Submitted Form below. You should see element/value pairs that match the data you entered in the form.

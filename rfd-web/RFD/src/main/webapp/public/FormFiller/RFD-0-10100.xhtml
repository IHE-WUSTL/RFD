RFD 0-10100

Special Instructions


Description

This test requires that the Form Filler initiate an ITI-34 transaction with the following requirements:

  1. Pre-population data is null
  2. workflowData.formID: RFD 0-10100
  3. workflowData.encodedResponse: false, meaning the Form Filler is requesting the URL of a form that can be opened in a web browser
  4. workflowData.archiveURL: The URL of the Form Archiver simulator.
  5. workflowData.context: null
  6. workflowData.instanceID: null

  • The Form Filler submits the Request for Form (ITI-34) using this value for formID: RFD 0-10100
  • The Form Manager simulator returns the URL of a form.
  • The Form Filler submits the form to the Form Receiver simulator.
  • The test system evaluates the request for all required properties (both positive and negative).
  • The test system examines the Form Receiver simulator for successful submission of the form (did your web browser workâ€¦).
  • The test system examines the Form Archiver simulator to verify that the form was archived.


Evaluation


Test Steps

  1. Submit a request for your form to the FormManager Simulator located at http://host:3205/RookieFormManager/. Be sure that your request follows the guidelines specified above in the Description.
  2. If your request has been formatted properly the Form Manager simulator should either return a URL of the form or a response saying that the requested form has not been found.
  3. If you received a URL, render the form. Complete the form and submit. The form should be transmitted to the Form Receiver simulator.
  4. To ensure the form was processed correctly, use the evaluate form request below to check all required properties.(both positive and negative).
  5. To check if the form was successfully submitted, use the evaluate retrieve form below.
  6. To check if the form was successfully archived, use the evaluate archive form below.
